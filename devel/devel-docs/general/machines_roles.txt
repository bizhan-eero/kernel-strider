Роли машин

Здесь приведено распределение ролей машин, на которых могут крутиться
компоненты Kernel Strider. Для каждой роли приведено краткое описание и
список компонентов Kernel Strider, которые должны загружаться на машине
с этой ролью.

Одна машина может выполнять несколько ролей, но (предполагаем) одна роль
не может распределяться между несколькими машинами.

Данное распределение может использоваться при конфигурации пакета,
предназначенного для машины с той или иной ролью(ролями).


1. Machine Under Analysis (MUA)

На этой машине запущен целевой модуль под управлением Kernel Strider.

Компоненты:
 - Core (выполняет инструментирование целевого модуля)
 - Event Collector (собирает данные полученные в ходе выполнения целевого модуля)
 - Event Sender (пересылает данные, собранные Event Collector'ом, по сети)


2. Trace Aggregator(TA)

На этой машине запущен сервер, который собирает с MUA трассу(трассы) и сохраняет ее в виде коллекции файлов.

Компоненты:
 - Event Receiver(принимает данные, посланные Event Sender'ом по сети,
   записывает их в файлы)


3. Analyzer

На этой машине выполняется анализ трассы (возможно, после ее предварительного преобразования для конкретных инструментов).

Компоненты:
 - TSan Converter (читает трассу из файлов, пишет один файл, который понимает  Thread Sanitizer)

На данной машине работает и Thread Sanitizer (выполняет анализ преобразованной трассы на предмет data race'ов).

 
 TCP Tunnel.
 Если используется TCP туннель для надежной пересылки данных между разными машинами, то одна часть этого туннеля находится на машине-MUA, другая на 
машине-TA.
